A request és a response

- Címsorba beírjuk az URL-t, pl. hwsw.hu

- Ilyenkor domainnév lekérés történik

- Elküldünk egy kérést (requestet)

- ezzel megy egy head és egy body

- a requestben van az URL

- a requestet a node.js-szel végezzük el

- ezt válaszküldés (response) követi

- a response egy csomag

- ebben is van head és body

- a bodyban van html, jpg, mov, stb.

- a renderelés a megjelenítést intézi

- ami a háttérben, a kiszolgálón megy, az a backend

- a node.js egy javascript-keretrendszer


Javascript-keretrendszerek közti különbségek

- A javascript-keretrendszerek közti különbségek kliens javascripten az interaktivitásért felelnek, és megvalósítanak üzleti logikát is.

- A backend keretrendszer csak az üzleti logikáért felel, a háttérben futó interaktív folyamatokat helyezzük ide.

- A backend keretrendszer képes külső adatforrásokhoz való hozzáféréshez.

- A node.js képes fájlt írni és olvasni, valamint adatbázist manipulálni.


URL

- U: egységes

- R: erőforrás

- L: leíró


- az URL központi része a szerver címe.


https://hwsw.hu:port/path#?

- hwsw.hu: domainnév.

- minden szerver erőforrás „//”-rel kezdődik.

- „:” elválasztó karakter

- http: protokollt jelent. Alapértelmezett portcím kapcsolódik hozzá. (A http-é 83.)

- Mindegyik protokollhoz tartozik egy alapértelmezett portcím.

- Ha szerver más porton várja a kérést, akkor a „:” után meg kell adni a portcímet.

- →path: helyi erőforrások, konkrét állományok, mappák megadására szolgál.

- „#”: az erőforrás tartozéka.

- „?”: az URL része, utána kulcsértékpárok vannak, amivel az erőforrást még jobban szűkíthetjük, az egyes értékpárokat „&”-sel választjuk el

- a request ezeket tartalmazza, a node.js alkalmazáson ezekkel dolgozik.

- A node.js-es munka nagy része szerverprogramozás.

- az RFC 7231 egy protokoll.

A REQUEST METÓDUSAI

- get: kérés a klienstől

- head: kérés a klienstől, csak a head, body nélkül

A kérdések URL-ként továbbítódnak, a többi kérés nem.

- post: kérés a klienstől a szerver állapotának megváltoztatására

- delete: kérés a klienstől a szerveren kijelölt entitás törlésére

- patch: kérés a klienstől a szerveren lévő adat átmeneti módosítására

- put: kérés a klienstől a szerveren lévő entitás felülírására


STATUS CODE

- 1 xy (100-as): információ.

- 2 xy (200-as): siker.

- 3 xy: átirányítás.

- 4 xy: kliens hiba.

- 404: ~

- 5 xy: szerverhibák.
